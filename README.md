# ЛК — система управления лицевыми счетами

React-приложение + бэкенд + Telegram-бот для управления ЛК, статусами и запросами «поднять с отдыха».

## Деплой на Railway

1. Подключите репозиторий к Railway.
2. Добавьте переменные окружения:
   - **BOT_TOKEN** — токен бота от [@BotFather](https://t.me/BotFather)
   - **TELEGRAM_CHAT_ID** — ID группы (например `-1001234567890`), можно также задать в админке
   - **DATA_DIR** — путь к Volume для сохранения данных (рекомендуется `/data`)
3. Подключите **Volume** к сервису и смонтируйте в `/data` — иначе данные сбросятся при редеплое.
4. Railway соберёт (`npm run build`) и запустит (`npm run start`).

Данные сохраняются в `server/data/` (локально) или в Volume на Railway: пользователи, ЛК, запросы, логи.

## Telegram-бот

Бот отправляет уведомления в группу при действиях:

- 🛑 Банкир поставил/снял **стоп** (ФИО + телефон)
- 📋 Банкир отметил **в вайте** / не в вайте
- 🔔 Оператор запросил **разблок** / **снятие Face ID** (с суммой застрявших средств)
- 🏦 Банкир ответил на запрос (да/нет + причина отказа)

Добавьте бота в группу и укажите ID чата в админке.

## Локальная разработка

```bash
npm install
npm run dev
```

Фронтенд: http://localhost:3000 (Vite). API и бот при локальной разработке не запускаются (нужен отдельный запуск `npm run start` после `npm run build`).

## Запуск с ботом (как на Railway)

```bash
npm run build
npm run start
```

Подставь в переменные окружения свой `BOT_TOKEN` (или создай файл `.env` с `BOT_TOKEN=...`).

## Функционал

- Роли: оператор (user), банкир, админ.
- **Банкир:** стоп реквизита, вайт/не в вайт, ответы на запросы операторов, поднятие с отдыха.
- **Оператор:** запрос разблока / снятия Face ID (с суммой застрявших средств).
- ЛК: CRUD, оборот, вкладки «Скоро на отдых», «На стопе», «В вайте», «Запросы операторов».
- Данные синхронизируются между всеми устройствами через сервер (каждые 5 сек).
